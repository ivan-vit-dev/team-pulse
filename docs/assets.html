Logo concept direction: a football with a pulse/heartbeat line running through it — the "pulse" of a team's season. Rendered in the navy→teal gradient with an optional floodlight-gold spark at the pulse peak. Flat, bold, geometric — no photographic imagery, no stadium clichés. It must not resemble any real club crest (teams upload their own crests next to it). Wordmark in Roboto Condensed Bold to match the in-app font-impact display face.

Definitive Color Palette

"Matchday, under lights." All tokens are oklch() — full definitions in APP_DESIGN.md / src/app/globals.css. Assets must work in BOTH light and dark themes. The floodlight gold is reserved for glow/highlight moments — it is not a general accent.

Each has a -foreground variant tuned to clear WCAG AA when used as text over its own ~18% tint — see the token table in APP_DESIGN.md. Use the base tone for solid markers only, never for text.

Brand Gradient

135deg, navy → teal — CTAs, logo, gradient text. Never introduce violet (hue ~285) — that belongs to a sibling project; and don't repurpose the floodlight gold as a warning/amber semantic color.

Team-brand override: teams carry their own colors, applied only inside team-scoped surfaces via --team-primary/--team-secondary. Brand assets must never hardcode any specific team's colors, and the app chrome always stays navy/teal.

Export Guidelines

SVG requirements

  • Flatten all text to paths — no font embedding
  • Optimize with SVGO; viewBox, not fixed width/height
  • Illustrations ≤ 8 kB, icons ≤ 2 kB after optimization
  • Both themes: use currentColor for strokes/neutral fills — the app ships light AND dark, and every asset renders on both
  • Accent colors: hardcode the oklch brand values (they're theme-stable enough), or ship -light/-dark pairs where they aren't

PNG requirements

  • Export at @2x minimum for marketing images
  • Compress with TinyPNG/Squoosh — marketing < 200 kB, OG < 300 kB
  • Apple touch icon: PNG-24, no transparency
  • Push badge: pure white on transparent (Android tints it)

Delivery format

  • Figma file with all assets on labeled frames
  • Zip export: /svg/, /png/, /ico/ folders
  • File naming: lowercase, hyphens, no spaces
  • Everything lands in public/ per the paths above

Testing checklist

  • Favicons in Chrome, Firefox, Safari, Edge — light and dark OS themes
  • OG images via opengraph.xyz or Meta's sharing debugger
  • Push icon: trigger a real FCM push from the emulator suite
  • Placeholders: create a youth player and a logo-less team, verify fallbacks
  • Empty states: fresh account walkthrough (no follows → no teams → new season)
Total estimated design effort: P1 (~half a day: logo + favicon pack via a generator + youth placeholder + OG image). P2 (~1 day: light variant, placeholders, two illustrations, push icon). P3 (~1–2 days). The logo, the youth-safe placeholder, and the OG image are the true blockers — everything else can ship as a second pass.
